The Science Behind the Squeeze

From pixels to matrices, explore the linear algebra that powers smart image compression.

A digital image is fundamentally a grid of pixels. For a color image, this grid is actually three-dimensional. It can be represented as a large matrix where each entry corresponds to a pixel's color. This color is typically defined by three intensity values for Red, Green, and Blue (RGB), each ranging from 0 to 255. Therefore, a color image of size $H \times W$ (Height x Width) can be seen as three separate $H \times W$ matrices, one for each color channel.

Singular Value Decomposition (SVD) is a powerful matrix factorization technique from linear algebra. It states that any matrix $A$ can be broken down into three other matrices: $$ A = U \Sigma V^T $$

  • $U$: An orthogonal matrix whose columns (left singular vectors) describe the spatial patterns in the image.
  • $\Sigma$: A diagonal matrix containing the singular values. These values are positive, sorted in descending order, and represent the "importance" of the information in the corresponding vectors.
  • $V^T$: The transpose of an orthogonal matrix whose rows (right singular vectors) describe feature patterns.

The magic of SVD for compression lies in the $\Sigma$ matrix. The largest singular values correspond to the most significant parts of the image, while the smaller ones often represent noise or fine details. By keeping only the top $k$ singular values and their corresponding vectors, we can create a very good approximation of the original image: $$ A \approx U_k \Sigma_k V_k^T $$ This approximation requires storing far less data than the original matrix $A$, thus achieving compression.

  1. Decomposition: The uploaded image is separated into its Red, Green, and Blue channel matrices. SVD is performed on each of these three matrices independently.
  2. Reduction (Approximation): Based on your selected quality setting ($k$), we keep only the top $k$ singular values and vectors for each channel, discarding the rest. A lower $k$ means more compression but lower quality.
  3. Reconstruction: The three new, smaller matrices for each channel ($U_k, \Sigma_k, V_k^T$) are multiplied together to reconstruct the compressed Red, Green, and Blue channels. These are then combined to form the final compressed image.
  4. Evaluation: We calculate the Compression Ratio and the Peak Signal-to-Noise Ratio (PSNR) to show you how much smaller the file is and how well it retained its quality compared to the original.
