How It Works?

What is a Digital Image?

A digital image is a representation of a visual scene using a grid of pixels. Each pixel is a tiny square that holds color information, typically defined by red, green, and blue (RGB) values. For example, a 1920x1080 image contains 2,073,600 pixels, and each pixelâ€™s RGB values (0-255) determine its color, resulting in a large amount of data.

Why Compress Images?

Storing and transmitting large image files can be inefficient, especially for web use or limited storage devices. Image compression reduces file size by removing redundant or less important data, balancing quality and storage needs.

How Does SVD Work?

Singular Value Decomposition (SVD) is a mathematical technique from linear algebra that breaks down a matrix into three components: U, S, and V. For images, each color channel (R, G, B) is treated as a matrix. SVD decomposes it as:

  • U: Left singular vectors (spatial patterns).
  • S: Singular values (importance of each pattern, sorted in descending order).
  • V: Right singular vectors (temporal or feature patterns).

By keeping only the top k singular values and their corresponding vectors, we approximate the original matrix, reducing data while preserving key features. The k value you select determines the trade-off between compression ratio and image quality.

The compressed image is reconstructed using: Compressed = U[:, :k] @ diag(S[:k]) @ V[:k, :], where only the most significant information is retained.

PixelSqueeze Process

PixelSqueeze applies SVD to each RGB channel of your uploaded image. You choose the k value, and the app:

  1. Decomposes the image into U, S, and V matrices.
  2. Keeps the top k singular values for compression.
  3. Reconstructs the image, calculating metrics like compression ratio and PSNR (Peak Signal-to-Noise Ratio) to measure quality loss.
