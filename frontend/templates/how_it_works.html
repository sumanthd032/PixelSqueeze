How PixelSqueeze Works?

What is a Digital Image? +

A digital image is a grid of pixels, each representing a tiny dot of color. These pixels use RGB values (0-255 for red, green, blue) to define colors, creating a detailed visual. For instance, a 1920x1080 image has over 2 million pixels, leading to large data sizes.

Why Compress Images? +

Compression reduces file size by eliminating redundant data, making images easier to store and share, especially for web or mobile use, while maintaining acceptable quality.

How Does SVD Work? +

Singular Value Decomposition (SVD) breaks a matrix (e.g., an image channel) into three parts:

  • U: Left singular vectors (spatial patterns).
  • S: Singular values (importance, sorted descending).
  • V: Right singular vectors (feature patterns).

By keeping only the top k singular values, we approximate the matrix, reducing data. The reconstruction uses: U[:, :k] @ diag(S[:k]) @ V[:k, :], balancing quality and size based on your chosen k.

PixelSqueeze Process +

PixelSqueeze applies SVD to each RGB channel:

  1. Decomposes the image into U, S, and V.
  2. Retains the top k values for compression.
  3. Reconstructs the image, computing compression ratio and PSNR.
